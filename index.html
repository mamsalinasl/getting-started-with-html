Introduction

Think Like a Programmer by using:

1. Procedural Thinking to tell the computer what to do using clear instructions.
2. Abstract Thinking to find similarities amongst different things.
3. Systems Thinking to help break down a big problem into smaller pieces.
4. Technological Empathy which means to see eye to eye with the computer; see how it sees things and both its advantages and limitations.
5. Debugging to figure out what is failing, why it’s failing, and how to fix it.

Unit I: The Basics of the Web

You & Your Computer (+Browser) --> The Internet --> HTTP --> Servers

Vocabulary:
• HTTP: the main protocol of the web
• Servers: computers that host the files that make up the web
• Internet: the world’s largest computer network
• Browser: a program that runs on your computer to display files found on the web

The World Wide Web
The world-wide web (www) is a collection of HTML documents.
Hyperlinks connect the sites together via URLs (links)
Anything can exist on the web.

HTML - HyperText Markup Language
The basis of most websites. It's a language used to give computers commands as to how we want websites to act and look like. It's made up of what we see, markup (what it looks like), links or references to other things (e.g. videos, images).

HTML Markup
I added extra spaces so the different tags could be displayed here and not be read as commands by the computer. Sorry if it makes it hard to understand!

< n a m e > contents < / n a m e >
Tags:
bold < b> < /b>
italic < em> < /em>
underline < u> < /u>


HTML Attributes
< Tag attr = "value" > content < / Tag>
< a > anchor tag
< a href = "www.google.com" > text goes here < / a>
< img src = "url" alt = "text" > VOID TAG (no content so don't need closing tag)


White Space
All white space in your working space turns into a single space.
< br > - break (void); inline (ending in a line)
< p > < / p > - paragraph; has content inside. Makes an invible box (a block). A paragraph can be stylized.

Inline Elements: < a>, < br>, < em>, img, strong, span
The inline elements stay along the line and just work to modify along it. The block elements have an imaginary box workspace that can be edited.

Span/Div:
Contain their content.
Span is inline; CSS
< span class = "foo" >
text
< / span >

Div - block that acts a bit like a paragraph
< div class = "bar" >
text
< / div >


HTML Documents:
There are different kinds of HTML documents. Here we're using HTML5:

< ! DOCTYPE HTML > - doctype is the type of HTML (currently HTML5)
< html > HTML type
First comes the head.
< head > Metadata goes here. This includes JS and CSS
< / head>
After the head comes the body. This is the actual visible content of the site.
< body >
Content goes here!
< /body >
< / html>
